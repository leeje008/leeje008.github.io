## 프로젝트 개요

블루오션 키워드 발굴 → 이미지 기반 초안 생성 → 네이버 D.I.A.+ SEO 최적화 → 원클릭 발행까지 전 과정을 자동화하는 1인 사용자용 블로그 도구이다. 로컬 LLM(Ollama)을 사용하여 API 비용 없이 운영할 수 있다.

## 해결한 핵심 문제들

### 문제 1: 경쟁이 낮은 키워드를 어떻게 찾을 것인가

블로그 상위 노출의 핵심은 "검색 수요는 있지만 경쟁이 적은" 블루오션 키워드를 찾는 것이다. 수동으로 키워드를 탐색하면 시간이 오래 걸리고 감에 의존하게 된다.

**해결: 네이버 자동완성 + LLM 확장 + 경쟁도 정량 분석**

```
시드 키워드 입력
    │
    ├── 네이버 자동완성 API 크롤링 → 실제 검색 수요가 증명된 키워드
    ├── LLM 롱테일 확장 → 15개 변형 키워드 생성
    └── 중복 제거 + 소스 태깅 (자동완성 우선)
    │
    ▼
블루오션 점수 산출 (0~100점)
```

**블루오션 점수 알고리즘** (3가지 지표의 가중합):

| 지표 | 배점 | 산출 방식 |
|------|------|----------|
| **소스 점수** | 30점 | 자동완성 출처 = 30점 (검색 수요 증명), LLM 생성 = 10점 |
| **상대 경쟁도** | 40점 | `seed_ratio = 타겟 블로그 수 / 시드 블로그 수` (로그 스케일) |
| **구체성** | 30점 | 시드 대비 추가 단어 수 (3개 이상 = 30점, 2개 = 25점) |

`seed_ratio`가 핵심이다. 예를 들어 "강남 카페"가 50,000건이고 "강남 카페 조용한 노트북"이 200건이면 `seed_ratio = 0.4%`로 블루오션에 해당한다. 단순 블로그 수가 아닌 **시드 키워드 대비 상대적 경쟁 수준**을 보는 것이 차별점이다.

### 문제 2: SEO 점수를 투명하게 진단하고 자동 개선할 수 있는가

네이버 검색에서 상위 노출되려면 D.I.A.+ 알고리즘의 기준을 충족해야 하는데, 이 기준은 공식적으로 공개되지 않는다. 블로거들은 감으로 글을 쓰고 결과를 확인할 수밖에 없었다.

**해결: D.I.A.+ 6개 지표 역공학 + 자동 개선 루프**

리서치를 통해 D.I.A.+ 알고리즘의 핵심 지표를 6개로 정리하고, 각 지표에 대해 정량적 검증 로직을 구현했다:

| 지표 | 가중치 | 검증 기준 |
|------|--------|----------|
| **제목** | 20% | 25자 이내, 키워드 앞쪽 배치 |
| **본문 길이** | 15% | 최적 2,500~3,000자 (최소 1,500자) |
| **키워드 밀도** | 25% | 1~2% 밀도, 본문 길이에 따라 5~6회 등장 |
| **제목 구조** | 20% | 1,000자당 H2 3~5개, H1→H2→H3 계층 준수 |
| **이미지** | 10% | 최소 3장(최적 6+), ALT 텍스트 20~50자에 키워드 포함 |
| **해시태그** | 10% | 5~10개, 키워드 포함 필수 |

추가 규칙:
- 키워드가 **첫 200자와 마지막 200자**에 반드시 등장해야 함
- H2 제목에 키워드 변형이 포함되어야 함
- 이미지 간격 300~500자 유지

등급 체계: A (90+), B (70~89), C (50~69), D (50 미만)

**SEO 자동 개선**: 검증 결과에서 점수가 낮은 항목을 피드백으로 LLM에 전달하면, 기존 문체를 유지하면서 해당 항목만 개선한 새 버전을 생성한다.

### 문제 3: 기존 블로그의 문체를 어떻게 유지할 것인가

LLM이 생성한 글은 기존 블로그 글과 톤이 다르면 독자에게 이질감을 준다. 하지만 매번 "이런 문체로 써줘"라고 지시하는 것은 일관성이 없다.

**해결: 레퍼런스 크롤링 + 프롬프트 주입**

```
기존 블로그 URL 3개 입력
    │
    ▼
BeautifulSoup 크롤링
    ├── 제목 + 본문 텍스트 (최대 1,500자)
    ├── 이미지 위치 배열 [paragraph_indices]
    └── JSON으로 저장 (data/references/)
    │
    ▼
초안 생성 시 system prompt에 주입
    ├── LLM이 문체 패턴 학습 (구어체, 문장 길이, 이모지 사용 등)
    └── 이미지 배치 패턴도 참조 → [IMAGE_N] 플레이스홀더 위치 결정
```

크롤러는 네이버 블로그의 DOM 구조 변경에 대비하여 **3단계 CSS 셀렉터 폴백**을 구현했다 (`.se-title-text` → `.pcol1` → `.se_title`). 크롤링이 실패하면 사용자가 직접 복사-붙여넣기할 수 있는 수동 입력 폴백도 지원한다.

SEO 최적화 단계에서도 레퍼런스가 다시 주입되어, 점수를 올리면서도 문체가 변하지 않도록 보장한다.

### 문제 4: 이미지를 포함한 글을 네이버에 자동 발행

네이버 블로그에 이미지가 포함된 HTML 글을 프로그래밍 방식으로 발행하려면 별도의 API 연동이 필요하다.

**해결: XML-RPC MetaWeblog API + Base64 이미지 임베딩**

```python
# MetaWeblog API 호출 구조
client.metaWeblog.newPost(
    blog_id, blog_id, api_secret,
    {
        "title": title,
        "description": html_with_images,
        "mt_keywords": "tag1,tag2,..."
    },
    True  # 즉시 발행
)
```

이미지 처리 파이프라인:
1. 포맷 검증 (JPEG/PNG/WEBP/GIF)
2. 최대 너비 860px 리사이즈 (네이버 블로그 최적)
3. RGBA → RGB 변환, JPEG 85% 품질
4. Data URI (Base64)로 HTML 내 인라인 삽입
5. `[IMAGE_N]` 플레이스홀더를 `<img src="data:image/jpeg;base64,..." />`로 치환

## 전체 워크플로우

Streamlit 5-페이지 구성:

```
[1. 설정] → [2. 키워드] → [3. 글쓰기] → [4. 미리보기] → [5. 히스토리]
    │           │              │              │
    │         시드 입력      이미지 업로드    SEO 점수 확인
    │         블루오션 분석  설명 입력        자동 개선
    │         타겟 선택      초안 생성(스트리밍)  원클릭 발행
    │
  Ollama 모델 선택
  네이버 API 인증
  레퍼런스 URL 크롤링
```

### LLM 응답 안정성 처리

LLM이 JSON 형식의 응답을 생성할 때 마크다운 펜스(` ```json `)를 포함하거나 불완전한 JSON을 반환하는 경우가 빈번하다. 이를 위해 **3단계 파싱 전략**을 구현했다:
1. 마크다운 펜스 제거
2. JSON 추출 (중괄호 범위 탐색)
3. 부분 JSON 복구 (trailing comma 제거 등)

## 기술 스택

| 분류 | 기술 | 선택 이유 |
|------|------|----------|
| UI | Streamlit (멀티페이지) | Python only, 빠른 프로토타이핑 |
| LLM | Ollama (로컬) | API 비용 0원, 오프라인 동작 |
| 크롤링 | BeautifulSoup + lxml | 네이버 DOM 파싱, 셀렉터 폴백 |
| 업로드 | XML-RPC MetaWeblog API | 네이버 공식 지원 API |
| 이미지 | Pillow | 리사이즈, 포맷 변환, Base64 |
| HTTP | ThrottledSession (0.5s) | 크롤링 레이트 리밋 방지 |
| 프롬프트 | YAML 템플릿 | 프롬프트와 코드 분리 |
| 패키지 관리 | uv | 빠른 의존성 해결 |
