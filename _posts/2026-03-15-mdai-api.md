## 프로젝트 개요

다중 사업부(9개 법인)를 지원하는 AI 기반 경영분석 챗봇 백엔드 시스템이다. LangGraph 에이전트와 Strategy 패턴을 결합하여 법인별 맞춤 분석(매출/원가/수익/스프레드)을 실시간 스트리밍으로 제공한다.

> 회사 프로젝트로 개인 레포지토리에는 소스 코드가 공개되어 있지 않습니다.

---

## 담당 역할

- 백엔드 API 설계 및 개발 (FastAPI)
- LLM 에이전트 파이프라인 구축 (LangGraph)
- 법인별 분석 전략(Strategy Pattern) 설계 및 구현
- RBAC V2 인증/인가 시스템 구축
- 테스트 코드 작성 (1,400+ 테스트 케이스)

## 시스템 아키텍처

```
Frontend (React)
    │
    ▼
FastAPI Backend (/api/v1)
    │
    ├── Auth Layer (JWT + RBAC V2)
    │   ├── 토큰 발급/갱신/무효화
    │   └── 권한 기반 접근 제어
    │
    ├── Conversation Layer
    │   ├── 대화 세션 관리
    │   ├── SSE 실시간 스트리밍
    │   └── 피드백 수집 (like/dislike)
    │
    ├── Analytics Layer
    │   ├── Strategy Factory (법인별 자동 선택)
    │   ├── LangGraph Agent (다단계 분석)
    │   └── 테이블/차트 생성기
    │
    └── Admin / HR Sync
        ├── 사용자/역할/권한 관리
        └── HR 시스템 동기화 (APScheduler)
    │
    ▼
PostgreSQL + ChromaDB (Vector)
```

## 핵심 기능

### 1. 법인별 맞춤 경영분석

Strategy 패턴 기반으로 9개 법인(RM오산, 하이원, 서울에코, 신풍, 미주, 대영, JS, RM화성, H2)에 대한 개별 분석 전략을 구현했다. Factory가 사업부 코드에 따라 자동으로 적절한 전략을 선택한다.

```
AnalyticsService
    │
    ▼
StrategyFactory.get_strategy(business_unit)
    │
    ├── RMOsanStrategy
    ├── HighOneStrategy
    ├── SeoulEcoStrategy
    └── ... (9개 법인)
```

분석 유형:
- 매출 분석 (Revenue)
- 원가 분석 (Cost)
- 수익 분석 (Profit)
- 스프레드 분석 (Spread)

### 2. LangGraph 멀티 에이전트

LangGraph 기반 상태 머신으로 다단계 분석 워크플로우를 구현했다. 각 노드가 데이터 조회, 분석, 테이블/차트 생성을 담당하며, SSE(Server-Sent Events)로 실시간 스트리밍 응답을 제공한다.

에이전트 유형:
- **Sorting Agent**: 주요 분석 (분류별 매출/원가)
- **MR Agent**: 원자재 재활용 분석
- **H2 Agent**: H2 특화 분석

### 3. RBAC V2 인증/인가

JWT 기반 인증에 세분화된 역할/권한 시스템을 결합했다.

- 역할(Role) → 권한(Permission) 매핑
- 권한 코드 기반 접근 제어 (예: `FEATURE_CHAT_USE`, `FEATURE_ADMIN_USER_MANAGE`)
- 감사 로그 (RBAC 변경 이력 추적)
- 토큰 갱신/무효화

### 4. 대화 시스템

- 대화 세션 관리 (생성/조회/삭제)
- 사용자 메시지 → AI 분석 → 구조화된 응답 (테이블/차트/후속 질문)
- 답변 재생성 기능
- 후속 질문(Follow-up) 지원

### 5. HR 시스템 연동

APScheduler 기반 주기적 HR 데이터 동기화로 사용자/부서 정보를 자동 반영한다.

## 주요 아키텍처 패턴

| 패턴 | 적용 영역 |
|------|----------|
| **Service Layer** | Router → Service → Repository → DB |
| **Strategy Pattern** | 법인별 분석 전략 자동 선택 |
| **Factory Pattern** | 전략 객체 자동 등록 및 캐싱 |
| **Prompt Injection** | 법인별 맞춤 프롬프트 제공 |
| **Repository Pattern** | BaseRepository 기반 공통 CRUD |
| **Dependency Injection** | FastAPI Depends 기반 인증/세션 |
